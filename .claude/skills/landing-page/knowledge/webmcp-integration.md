# WebMCP Integration Guide for Landing Pages

## What is WebMCP?
WebMCP is the official JavaScript library that turns any webpage into an MCP server. AI agents can discover and call tools on the page — click buttons, fill forms, read content — all through structured MCP protocol, not DOM scraping.

## MANDATORY: Every landing page MUST include WebMCP

All HTML/CSS/JS pages generated by the build agent MUST include the official WebMCP library. This is non-negotiable — it's the core of our agentic web strategy.

## Integration — Official WebMCP Library

Add this **before the closing `</body>` tag** in every generated HTML page:

```html
<!-- WebMCP Integration — Official Library -->
<script src="https://cdn.jsdelivr.net/npm/webmcp@latest/webmcp.js"></script>
<script>
  // Initialize WebMCP with branding
  const mcp = new WebMCP({
    color: '#4CAF50',
    position: 'bottom-right',
    size: '40px',
    padding: '15px'
  });

  // ── Register CTA / Link Tools ──────────────────────────────
  // Register each CTA, button, and important link as a tool
  document.querySelectorAll('a[toolname], button[toolname]').forEach(function(el) {
    var name = el.getAttribute('toolname');
    var desc = el.getAttribute('tooldescription') || 'Click: ' + el.textContent.trim();
    mcp.registerTool(name, desc, {}, function() {
      el.click();
      return { content: [{ type: 'text', text: 'Clicked: ' + name }] };
    });
  });

  // ── Register Form Tools ────────────────────────────────────
  document.querySelectorAll('form[toolname]').forEach(function(form) {
    var name = form.getAttribute('toolname');
    var desc = form.getAttribute('tooldescription') || 'Submit form';
    var schema = {};
    form.querySelectorAll('input, select, textarea').forEach(function(field) {
      if (field.type !== 'hidden' && field.type !== 'submit' && field.name) {
        schema[field.name] = { type: 'string', description: field.placeholder || field.name };
      }
    });
    mcp.registerTool(name, desc, schema, function(args) {
      Object.keys(args).forEach(function(key) {
        var input = form.querySelector('[name="' + key + '"]');
        if (input) input.value = args[key];
      });
      form.submit();
      return { content: [{ type: 'text', text: 'Form submitted: ' + name }] };
    });
  });

  // ── Register Section Read Tools ────────────────────────────
  document.querySelectorAll('section[id], [data-section]').forEach(function(section) {
    var sectionId = section.id || section.dataset.section;
    var heading = section.querySelector('h1,h2,h3');
    var name = 'read_' + sectionId;
    var desc = 'Read content: ' + (heading ? heading.textContent.trim() : sectionId);
    mcp.registerTool(name, desc, {}, function() {
      return { content: [{ type: 'text', text: section.textContent.trim() }] };
    });
  });

  // ── Register Page Context Resource ─────────────────────────
  mcp.registerResource(
    'page-context',
    'Full page context for AI agents',
    { uri: 'page://context', mimeType: 'application/json' },
    function() {
      return {
        contents: [{
          uri: 'page://context',
          mimeType: 'application/json',
          text: JSON.stringify({
            url: window.location.href,
            title: document.title,
            description: (document.querySelector('meta[name="description"]') || {}).content || '',
            sections: Array.from(document.querySelectorAll('section[id], [data-section]')).map(function(s) {
              return { id: s.id || s.dataset.section, heading: (s.querySelector('h1,h2,h3') || {}).textContent || '' };
            })
          })
        }]
      };
    }
  );
</script>
```

## How It Works

1. **Blue widget** appears on the page (bottom-right corner)
2. **AI agents** connect via any MCP client → widget → paste MCP token
3. **Tools are discovered** — every CTA, form, and section is a callable tool
4. **Resources exposed** — page context available as structured data

## How 3rd Party AI Agents Connect

Any AI agent (Claude, ChatGPT, Cursor, etc.) can interact with a WebMCP-enabled page:

1. Add WebMCP to their MCP client config:
```json
{
  "mcpServers": {
    "webmcp": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/webmcp@latest", "--mcp"]
    }
  }
}
```

2. Open the landing page in browser
3. Click the blue WebMCP widget → paste MCP token from their client
4. Their AI now sees all registered tools and can interact with the page

## Build Agent Instructions

When generating HTML, the build agent MUST:

1. Add `toolname` and `tooldescription` attributes to ALL:
   - CTA buttons and links
   - Forms
   - Interactive elements

2. Add `id` and `data-section` attributes to ALL sections

3. Load the official WebMCP library via `<script src="...webmcp.js"></script>`

4. Register all tools using `mcp.registerTool()` after loading the library

Example:
```html
<section id="hero" data-section="hero">
  <h1>Headline</h1>
  <p>Subheadline</p>
  <a href="/signup"
     toolname="signup_cta"
     tooldescription="Sign up for the product"
     class="cta">Get Started</a>
</section>

<section id="pricing" data-section="pricing">
  <h2>Pricing</h2>
  <!-- pricing content -->
</section>

<form id="lead-capture"
      toolname="capture_email"
      tooldescription="Submit email to join waitlist">
  <input type="email" name="email" required placeholder="Enter your email" />
  <button type="submit" toolname="submit_email" tooldescription="Submit email form">
    Join Waitlist
  </button>
</form>
```
