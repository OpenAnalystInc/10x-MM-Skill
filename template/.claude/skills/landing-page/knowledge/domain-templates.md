# Domain Templates — Landing Page Categories

Every landing page serves a different purpose depending on the domain. A SaaS product page needs feature comparisons and pricing, while a portfolio page needs project showcases and contact forms. This file defines the correct structure for each domain.

---

## HOW TO USE THIS FILE

1. Identify the user's domain from their input (or ask)
2. Load the matching template below
3. Use ONLY the sections relevant to that domain
4. Skip sections marked [OPTIONAL] unless user explicitly requests them
5. Adapt the page structure, data points, and CTAs to match the domain

---

## DOMAIN 1: SaaS / SOFTWARE PRODUCT

**Purpose**: Convert visitors to free trial, demo, or signup
**Primary CTA**: Start Free Trial / Get Started / Book Demo

### Required Sections (in order)
1. **Hero** — Headline (outcome-focused), subhead (how it works), CTA button, product screenshot/demo
2. **Social Proof Bar** — Logo strip of notable customers OR "Trusted by X,000+ teams"
3. **Problem Statement** — Paint the pain point (before state)
4. **Solution Overview** — 3-4 key features with icons + short descriptions
5. **How It Works** — 3-step process (Sign up → Configure → Get Results)
6. **Feature Deep Dive** — 3 detailed features with screenshots (alternating left/right layout)
7. **Social Proof** — 2-3 customer testimonials with photo, name, role, company
8. **Pricing** — 2-3 tiers (or "Start free, upgrade anytime")
9. **FAQ** — 5-7 common objections answered
10. **Final CTA** — Repeat primary CTA with urgency

### Data Points Needed
- Product name, tagline
- 3-4 key features with descriptions
- Pricing tiers (or free trial details)
- Customer logos, testimonials
- Product screenshots

### Intelligent Routing
- **Always invoke**: Copy, Design, Build
- **Auto-invoke if commercial**: SEO, Analytics
- **Invoke if user mentions**: Speed, A/B Test, Lead Capture
- **Skip unless asked**: Competitor, Content Strategy, Funnel

## DOMAIN 2: ECOMMERCE / PRODUCT PAGE

**Purpose**: Convert visitors to purchase
**Primary CTA**: Add to Cart / Buy Now / Shop Now

### Required Sections (in order)
1. **Hero** — Product image (large, high-quality), product name, price, star rating, CTA
2. **Product Gallery** — Multiple images, zoom on hover
3. **Product Details** — Specifications, materials, dimensions, variants (size/color)
4. **Benefits** — 3-4 key benefits (not features — what customer gets)
5. **Social Proof** — Customer reviews with star ratings, verified purchase badges
6. **Trust Signals** — Free shipping, returns policy, secure payment badges, warranty
7. **Cross-Sell** — "You might also like" or "Frequently bought together"
8. **FAQ** — Shipping, returns, sizing questions

### Data Points Needed
- Product name, price, variants
- Product images (multiple angles)
- Specifications / materials
- Customer reviews
- Shipping/returns policy

### Intelligent Routing
- **Always invoke**: Copy, Design, Build
- **Auto-invoke**: SEO (product schema), Analytics (ecommerce events), Speed (image-heavy)
- **Invoke if user mentions**: A/B Test (pricing/CTA), Lead Capture (email for back-in-stock)
- **Skip unless asked**: Funnel, Competitor, Content Strategy

---

## DOMAIN 3: PORTFOLIO / PERSONAL BRAND

**Purpose**: Showcase work, attract clients/employers
**Primary CTA**: Contact Me / Hire Me / View My Work

### Required Sections (in order)
1. **Hero** — Name, title/tagline, professional photo, CTA
2. **About** — Brief bio (2-3 sentences), key skills/specialties
3. **Featured Work** — 3-6 project cards (image, title, brief description, link)
4. **Skills/Services** — What you offer (grid or list)
5. **Testimonials** — 2-3 client quotes [OPTIONAL]
6. **Contact** — Contact form or email link + social links

### Data Points Needed
- Name, title, photo
- 3-6 portfolio pieces with images
- Skills list
- Contact info, social links

### Intelligent Routing
- **Always invoke**: Copy (minimal), Design, Build
- **Auto-invoke**: SEO (basic meta tags)
- **Skip by default**: Analytics, Speed, A/B Test, Funnel, Leads, Competitor, Content
- **This is a lightweight page** — skip heavy optimization unless asked

---

## DOMAIN 4: IT SUPPORT / TECH SERVICES

**Purpose**: Generate leads for IT services
**Primary CTA**: Get a Free Consultation / Request Quote / Call Now

### Required Sections (in order)
1. **Hero** — Headline (solve IT pain), subhead, CTA, trust badge (certified/years)
2. **Pain Points** — 3-4 common IT problems the audience faces
3. **Services** — Grid of services offered (network, security, cloud, helpdesk, etc.)
4. **Why Choose Us** — Differentiators (response time, certifications, experience)
5. **Process** — How engagement works (Assessment → Plan → Implement → Support)
6. **Case Studies** — 2-3 results (before/after metrics)
7. **Trust Signals** — Certifications (CompTIA, Microsoft, Cisco), years, client count
8. **Testimonials** — Business owner quotes
9. **Service Areas** — Geographic coverage [OPTIONAL]
10. **Contact/Quote Form** — Name, email, company, issue type, message

### Data Points Needed
- Service list with descriptions
- Certifications, years in business
- Client count, case study metrics
- Service area / geographic info

### Intelligent Routing
- **Always invoke**: Copy, Design, Build, Leads (quote form is critical)
- **Auto-invoke**: SEO (local SEO important), Analytics
- **Invoke if user mentions**: Speed, Content Strategy
- **Skip unless asked**: A/B Test, Funnel, Competitor, JS Injection

## DOMAIN 5: EVENT / WEBINAR / CONFERENCE

**Purpose**: Drive registrations
**Primary CTA**: Register Now / Save Your Spot / RSVP

### Required Sections (in order)
1. **Hero** — Event name, date/time, location (or "Virtual"), CTA, countdown timer
2. **Event Description** — What attendees will learn/experience
3. **Speakers/Hosts** — Photo, name, title, brief bio for each
4. **Agenda** — Timeline of sessions/topics
5. **Benefits** — What attendees take away (3-5 bullet points)
6. **Social Proof** — Past event photos, attendee count, testimonials from past events
7. **Pricing** — Ticket tiers (early bird, regular, VIP) [OPTIONAL for free events]
8. **FAQ** — Logistics questions (parking, virtual access, recording, refunds)
9. **Registration Form** — Name, email, company [OPTIONAL: dietary, t-shirt size]
10. **Urgency** — "Only X spots left" or "Early bird ends [date]"

### Data Points Needed
- Event name, date, time, location
- Speaker details
- Agenda/schedule
- Ticket pricing (if paid)

### Intelligent Routing
- **Always invoke**: Copy, Design, Build, Leads (registration form)
- **Auto-invoke**: SEO (event schema), Analytics (registration tracking)
- **Auto-invoke**: Inject (countdown timer JS)
- **Skip unless asked**: Speed, Funnel, Competitor, Content

## DOMAIN 6: DEMO / PRODUCT DEMO

**Purpose**: Get prospects to book a product demo
**Primary CTA**: Book a Demo / Schedule a Call / See It in Action

### Required Sections (in order)
1. **Hero** — Headline (see how [product] solves [problem]), CTA, demo video thumbnail or product screenshot
2. **Demo Video** — Embedded video or GIF walkthrough [OPTIONAL if no video]
3. **Key Benefits** — 3-4 outcomes the prospect will see in the demo
4. **How It Works** — Quick 3-step overview
5. **Who It's For** — Ideal customer profiles
6. **Social Proof** — Customer logos + 1-2 short testimonials
7. **Booking Calendar** — Embedded Calendly/Cal.com or simple form (name, email, company, time preference)
8. **FAQ** — Demo length, what to expect, no commitment required

### Data Points Needed
- Product name, demo video URL
- Calendly/booking link
- Target audience descriptions
- 2-3 key outcomes

### Intelligent Routing
- **Always invoke**: Copy, Design, Build, Leads (booking form)
- **Auto-invoke**: Analytics (demo booking events), Inject (calendar embed)
- **Skip by default**: SEO (usually paid traffic), Speed, A/B Test, Funnel, Competitor

## DOMAIN 7: LEAD MAGNET / RESOURCE DOWNLOAD

**Purpose**: Capture email in exchange for free resource
**Primary CTA**: Download Free [Resource] / Get Your Copy / Access Now

### Required Sections (in order)
1. **Hero** — Resource title, what they'll learn, CTA + email form, resource mockup image
2. **What's Inside** — 3-5 bullet points of what the resource covers
3. **Who It's For** — Target audience
4. **Author/Source Credibility** — Brief bio or company credibility
5. **Social Proof** — "Downloaded by X,000+ people" or testimonials
6. **Sample Preview** — 1-2 pages/sections shown as teaser [OPTIONAL]
7. **Final CTA** — Repeat form with urgency

### Data Points Needed
- Resource title, format (PDF, video, template)
- 3-5 key takeaways
- Author/company name
- Download count (if available)

### Intelligent Routing
- **Always invoke**: Copy (headline critical), Design, Build, Leads (email capture)
- **Auto-invoke**: Analytics (download tracking)
- **Skip by default**: SEO, Speed, Funnel, Competitor, Content, A/B Test

## INTELLIGENT ROUTING RULES

### How to determine which skills to auto-invoke:

**Parse the user's input for these signals:**

| Signal in user input | Skills to auto-invoke |
|---|---|
| "with SEO" / "SEO optimized" / "search engine" | → SEO |
| "with analytics" / "tracking" / "GA4" / "pixel" | → Analytics |
| "fast" / "performance" / "speed" / "Core Web Vitals" | → Speed |
| "A/B test" / "variants" / "split test" | → A/B Test |
| "lead capture" / "email capture" / "popup" / "form" | → Leads |
| "funnel" / "upsell" / "email sequence" / "drip" | → Funnel |
| "competitor" / "vs" / "compared to" / "differentiate" | → Competitor |
| "inject" / "chat widget" / "Hotjar" / "Intercom" / "pixel" | → Inject |
| "content strategy" / "blog" / "testimonials" / "case study" | → Content |
| "audit" / "review" / "check" / "score" | → Audit |
| "optimize" / "conversion" / "CRO" / "improve" | → Optimize |
| "copy" / "rewrite" / "headlines" / "CTA" | → Copy |
| "design" / "colors" / "typography" / "visual" | → Design |

### Token-saving rules:
1. **Minimal request** (e.g., "create a portfolio page"): Only invoke Copy + Design + Build
2. **Standard request** (e.g., "create a SaaS landing page"): Follow domain's "Always invoke" + "Auto-invoke" list
3. **Full request** (e.g., "create a landing page with everything — SEO, analytics, testing"): Invoke all mentioned skills
4. **Never invoke skills the domain marks as "Skip"** unless user explicitly requests them
